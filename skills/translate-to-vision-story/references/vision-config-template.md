# Vision (Template)

> このテンプレートを `~/.claude/skills-config/vision.md` にコピーして自分のビジョンで書き換える。
> ビジョン要素は判定可能な粒度に分解すること (`translate-to-vision-story` skill が機械的に整合判定する)。

## ビジョン宣言

(自分の 1-3 行の宣言文を書く。例:)
非同期ドリブンで壊さず速くする。設計書とAI基盤で同期コストを最小化し、チームの開発速度を引き上げる。

## ビジョン要素 (判定基準)

| ID | 要素 | 説明 |
|---|---|---|
| V1 | (要素名) | (1-2 行の判定基準) |
| V2 | ... | ... |

### 分解ガイダンス

宣言文を「**判定可能な要素**」に分解するコツ:

1. 動詞単位で切る (例: 「壊さず速くする」→「壊さず」「速く」)
2. 道具と目的を分ける (例: 「設計書とAI基盤で同期コスト最小化」→「設計書駆動」「AI基盤活用」「同期コスト最小化」)
3. 個人軸とチーム軸を分ける (例: 「チームの開発速度を引き上げる」→「チーム速度向上」)
4. 5-10 項目に収める (多すぎると判定がぼやける、少なすぎると粒度が粗い)

### 例 (Yasuaki さんのケース)

| ID | 要素 | 説明 |
|---|---|---|
| V1 | 非同期ドリブン | チーム/自分との同期を強要しない仕組み |
| V2 | 壊さず | 既存を破壊せず安全に変更 |
| V3 | 速く | 絶対速度・リードタイム短縮 |
| V4 | 設計書駆動 | Design Doc / ADR で意思決定の非同期化 |
| V5 | AI基盤活用 | Claude Code / skill / MCP の活用 |
| V6 | 同期コスト最小化 | mtg 数削減・対話の構造化 |
| V7 | チーム速度向上 | 個人ではなく集団のアウトプット拡大 |
