# Step 3 チェックリスト化の詳細ルール

SKILL.md の Step 3（プロパティ・チェックリスト化）から参照される詳細ルール。実際に Figma MCP へ接続して抽出しているときにだけ踏む、重複 atom の統合ルールと、値が構造化データから確定できないときの保留・引き直し手順をまとめる。

## 重複 atom を作らない

重複 atom を作らない: 同一の子要素で 2 カテゴリの状態が常に連動して確定する場合は 1 行に統合する (例: 唯一の子要素がアイコンそのもの → 「要素の有無」と「アイコン」をアイコン行 1 本に、テキスト単独の子要素が丸ごと欠落 → 「要素の有無」1 本に、icon-only 要素の「配置」は「アイコン」行の記述に畳み込む)。「状態」の複合行は複数プロパティが同時に変わる状態の取りこぼし防止用で、構成プロパティが個別行で既に atom 化されているなら省略してよい。1 カテゴリ内でサブ属性ごとに信号の有無が異なる場合 (例: 角丸だけ明示値があり太さ・色は沈黙) は、信号のあるサブ属性のみ atom 化し、無いサブ属性だけを個別に省略する (カテゴリ丸ごとの all-or-nothing にしない)。

## 値が構造化データから確定できないとき（保留表記し、断定しない）

- **ツールの沈黙を「指定なし」と断定しない**: `get_variable_defs` / `get_design_context` がプロパティを返さなくても「Figma に指定がない」とは限らない（共有 variable collection / 親ノード / コンポーネントマスタに定義されていることがある）。`get_design_context` がトークン名（例 `Surface/Default`）を示すのに `get_variable_defs` が hex を解決しない、のような食い違いは典型。「当該ノードのツール出力に不在（別管理の可能性・要確認）」と保留表記し、**親ノード ID やページ ID で `get_variable_defs` / `get_metadata` を引き直して**解決を試みる。沈黙を仕様の沈黙と同一視すると、そのプロパティが照合対象から落ちて反映漏れになる。
- **「Figma 指定なし」は現状維持が既定**: 指定が本当に無いプロパティは現状の実装値を維持する。勝手に削除・変更へ倒さない。他の確定差分（例: 固定幅化）と干渉する場合のみ「要確認」フラグを付けて実装判断へ送る。
- **概念値で記録し、CSS への写像は実装へ渡す**: 差分行は「Figma 概念値（例: 中央寄せ）→ 現状値」で記録する。Figma 属性 → CSS プロパティ（`text-align` か `justify-content` か flex 配下か等）の具体的な写像は実装 / AC 化フェーズの責務。抽出段で CSS プロパティを一意に確定しようとして止まらない（抽出は「何を満たすべきか」、写像は「どう書くか」と腑分けする）。
- **引き直し先は値の種類で変える**: 引き直すときは引き先を値の種類で選ぶ。色は共有色コレクション、タイポ / spacing / border は別コレクションやコンポーネントマスタにあることが多い。インスタンスノードは `get_metadata` の `mainComponent` 属性が指すマスタノード ID を起点に `get_variable_defs` を引く。1 つのコレクションで解決しなくても、未解決の種別は別コレクション / マスタを当たる（解決できなければ捏造せず「要確認」のまま残す）。値の種類と node 種別（instance か）が競合したら**値の種類を優先**する: 色はまず共有色コレクションを当たり、そこで解決しない種別だけ `mainComponent` マスタへフォールバックする。
- **要素の有無は `get_metadata` を網羅的列挙とみなす**: `get_metadata` の子要素列挙に現れない要素種別は「無（確定＝要素まるごとの欠落）」として扱い、欠落を確定検出する。一方、枠線・影などスタイルプロパティの沈黙（子要素ではない）は上記の保留（引き直し対象）に回す。**要素の不在（確定）とスタイルの沈黙（保留）を混同しない。**
- **沈黙の終端処理**: テキスト内容 / アイコンの有無が沈黙なら当該子ノード ID で `get_metadata` / `get_design_context` を 1 段深掘りして解決する。variant（hover/active 等の状態）や layout は当該ノード出力に出ないのが常態なので、既定で保留（要確認）扱いとし確定差分に積まない。引き直しても解決しない沈黙は、捏造せず「要確認（別管理の可能性）」ラベルで現状維持にし確定差分にしない（「Figma 指定なし＝現状維持」と終端の扱いは同じだが、沈黙は確定前に最低 1 回引き直す点が異なる）。
- **省略テキストは実文言を要確認**: `get_metadata` / `get_design_context` のテキストが `...` のような省略表記なら、文言の正確一致は実テキストで確認するため「要確認」フラグを付ける（ここでもスクショを正本にしない）。
