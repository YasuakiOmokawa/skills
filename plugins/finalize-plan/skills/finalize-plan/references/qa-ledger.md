# QA 実行台帳 (`<plan>.qa-ledger.md`)

finalize-plan の Step 4 が初期化する。実装フェーズ (skill 外、セッション主エージェントの既存運用) では進捗メモとして追記されるが、正本になるのは `/qa-ui` の審判が再実行ゲートで上書きした後の状態のみ。完了判定・auto 再実行ゲートは `/qa-ui` 側の責務であり、本ファイルは finalize-plan 側で守るべき初期化・書式仕様を扱う。

## 置き場・命名

プランファイルと同ディレクトリ、拡張子前に `.qa-ledger` を挿入する (例: `design_fix.md` → `design_fix.qa-ledger.md`)。

## フォーマット

追記型 (append-only) の Markdown テーブル:

```markdown
| QA-ID   | 手段   | 状態                     | ラウンド | 備考 |
|---------|--------|--------------------------|----------|------|
| QA-H-01 | manual | pending                  | 0        | -    |
| QA-E-01 | auto   | pending                  | -        | -    |
| QA-X-01 | -      | 対象外(N/A)              | -        | 担当手段未特定、要人間確認 |
```

ラウンド列は手段に揃える: manual 由来は実測ラウンド番号 (初期化時は 0)、auto 由来と対象外は `-`。`検証不能(真の制約)` 等へ遷移した後もこの規則のまま。

行は削除・上書きせず追記のみで更新する。**同一 (QA-ID, 手段) の組は最後の行が現在状態** (「最新行が勝つ」)。ある QA-ID の状態を知りたい場合は、その (QA-ID, 手段) の組で最後に現れた行を見る。

## 状態語彙 (これ以外を使わない)

| 状態 | 意味 | 完了集計での扱い |
|---|---|---|
| `pending` | 初期値、未実行 | 許容されない (未完了) |
| `PASS` | 合格 | 許容される |
| `FAIL(<重大度orexit>)` | 不合格。例: `FAIL(Major)` / `FAIL(exit=1)` | 許容されない (未完了、再検証待ち) |
| `検証不能(真の制約)` | automation では検証できない真の制約 (workaround なし) | 許容される (非ブロッキング) |
| `要人間確認` | 判定不能・担当手段未特定など安全側の分類 | 許容されない (部分完了レポートに列挙) |
| `対象外(N/A)` | 対象外。備考に理由必須 | 許容される |

完了 = 最新行集計で全行が `PASS` / `検証不能(真の制約)` / `対象外(N/A)` のいずれか。

## 手段割当規則 (Step 4 で適用)

1. auto-qa-planner の QA-ID カバレッジマトリクスに載っている QA-ID → `auto`
2. 1 に該当せず manual-qa-planner の見出し (`**QA-ID | 出典: ...**`) に載っている QA-ID → `manual`
3. 両方に載っている (dual coverage) → `auto` を正とし、manual 行は作らない (auto 側でカバー済みなのに manual 側にも pending 行が残って完了しない状態を防ぐ)
4. どちらにも載っていない QA-ID → `対象外(N/A)`、備考「担当手段未特定、要人間確認」(安全側のデフォルト分類)

初期化 Bash は SKILL.md の Step 4 参照 (fixture で実行検証済み)。

## 実装フェーズでの auto 行追記 (進捗メモ、暫定)

赤→緑ループで QA-ID が緑になった時点で、セッション主エージェントが以下の形式で auto 行を追記してよい:

```bash
echo "| QA-E-01 | auto | PASS | -        | 実装者の自己申告 (審判未検証) |" >> "<plan>.qa-ledger.md"
```

この記帳はあくまで暫定の進捗メモで、`/qa-ui` の審判が再実行ゲートで上書きするまで正本にならない。既存行は書き換えず追加のみで行う (「最新行が勝つ」規則により後続の追記が優先される)。
