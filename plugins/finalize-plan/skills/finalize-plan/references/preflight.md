# Preflight 契約 (`<プラン名>.preflight.md`)

ループ開始前に一括収集する入力の置き場。生成者は finalize-plan (SKILL.md の Step 5)。参照者は `/qa-ui` (Step 2 / Phase A・B) と `branch-planner` (起点ブランチ確認)。

## 置き場・命名

プランファイルと同ディレクトリ、拡張子前に `.preflight` を挿入する (例: `feature-xxx.md` → `feature-xxx.preflight.md`)。

## 項目表

| 項目 | 内容 | 記載範囲 |
|---|---|---|
| ベース URL | 検証対象環境の URL | qa-ui Step 2 が参照 |
| ログイン手段 | 誰が・いつ・どの URL でログインするかの手順 | 手順のみ。実際の認証情報は書かない。人間向け手順メモであり、qa-ui はこの欄を読まずログイン画面検出時は常に停止して人間に依頼する |
| 権限アカウント一覧 | 検証に必要な権限種別とその用途 | 権限名と用途のみ。実アカウント (メールアドレス等) は書かない。qa-ui Phase B が参照 |
| テストデータ準備手順 | 固有データ・状態を要する QA-ID/AC のための準備コマンド・手順 | qa-ui Phase A が参照 |
| 起点ブランチ | branch-planner が確定した起点ブランチ名 | branch-planner が参照 (qa-ui は読まない) |
| サーバ・DB 起動コマンド | 検証環境を起動するコマンド | 人間向け参考情報。qa-ui はこの欄を読んで自動起動せず、接続失敗時は現行どおり停止して人間に依頼する |

## セキュリティ規則 (厳守)

1. **パスワード・トークン等の秘密情報はこのファイルに書かない。** ログイン手段欄・権限アカウント一覧欄は手順・権限名・用途に限定し、実際の認証情報 (パスワード、API トークン、実アカウントのメールアドレス等) は書かない。
2. **ログイン操作は常にユーザーが行う。エージェントによる自動ログインは行わない** (2026-07-05 の利用者決定)。qa-ui の「ログイン操作はユーザーの責任、エージェントは自動ログインを行わない」原則 (SKILL.md Step 2) と整合させる。認証つき UI 案件では QA 開始時のログイン 1 タッチが正式にループへ残る。検証対象プロジェクト側に開発環境限定の認証バイパスがある場合はログイン手段欄にそれを書けばこのタッチは消えるが、これは任意でありエージェント側から提案しない。

## 未定項目の扱い

生成時に埋まらなかった項目は `未定` と記載する。finalize-plan は `未定` が残る項目をまとめて **1 回の AskUserQuestion** でユーザーに確認する (項目ごとに個別に停止しない)。既存の preflight ファイルを補完する場合も、不足項目のみをこの一括確認にまとめる。AskUserQuestion が利用可能ツールに無い実行文脈での読み替えは SKILL.md「## 委譲実行」参照。
