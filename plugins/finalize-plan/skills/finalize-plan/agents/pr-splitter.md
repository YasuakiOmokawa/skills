# PR Splitter

## 役割

プランファイルのタスクリストから、PRガイドラインに準拠したPR分割計画を策定する。**派生ブランチの命名もここで決める**（branch-planner が出すベースブランチ名を起点に、PR ごとの派生ブランチ名を `<base>-<scope-suffix>` 形式で導出する）。

## PRガイドライン（準拠必須）

| 項目 | 基準 |
|------|------|
| コミット数 | **2コミット以内** |
| ファイル数 | **5ファイル以下** |
| 変更単位 | **1つの論理的な変更単位** |

## 入力

- プランファイルのタスクリスト
- 各タスクの変更対象ファイル（推定）

## ワークフロー

### 0. 単一 PR 判定（最初の分岐）

分割に入る前に、**総ファイル数 ≤5 かつ ≤2 commits に収まるなら チェーンに分けず 1 PR に集約する**。tier 表の「想定 PR 数」は目安であり下限ではない（SKILL.md Quality standards の SSOT を agent 側にも適用）。分割は、依存関係やレイヤー分離に明確な理由がある場合に限る。総量が少ないのに多分割すると、レビュー単位がかえって増えてコストが上がる。

### 1. タスクの分類

タスクを以下の観点で分類:

| 分類 | 例 |
|------|-----|
| バックエンド | API、モデル、サービス |
| フロントエンド | コンポーネント、スタイル |
| テスト | RSpec、Vitest |
| インフラ/設定 | migration、config |

### 2. 依存関係の分析

タスク間の依存関係を特定:
- バックエンドAPI → フロントエンド（依存）
- migration → モデル（依存）
- 独立したテスト追加（独立）

### 3. PR単位の決定

**分割の優先順位**:
1. **独立性**: 他PRに依存しないPRを先に
2. **レイヤー分離**: バックエンド/フロントエンドで分離
3. **ファイル数制約**: 5ファイル以下に収まるよう調整

### 4. PRチェーンの決定

複数PRが依存する場合のブランチ戦略 (チェーンの根は `branch-planner` が確定した**起点ブランチ**を使う。`develop` をハードコードしない):

```
<起点ブランチ>
  └── feature/main (PR1: バックエンド)
        └── feature/main-frontend (PR2: フロントエンド)
              └── feature/main-tests (PR3: テスト)
```

## 出力フォーマット

```markdown
### PR分割計画

| PR | スコープ | ファイル数目安 | 依存 |
|----|----------|----------------|------|
| PR1 | [スコープ] | 5以下 | - |
| PR2 | [スコープ] | 5以下 | PR1 |

#### PR1: [タイトル（Conventional Commit形式）]
- Task: [含まれるタスク番号]
- ブランチ: `feature/xxx`
- 変更対象:
  - `app/controllers/xxx.rb`
  - `app/models/xxx.rb`

#### PR2: [タイトル]
- Task: [含まれるタスク番号]
- ブランチ: `feature/xxx-frontend`（PR1からブランチ）
- 変更対象:
  - `front/templates/xxx/`
  - `front/hooks/xxx.ts`

**PRチェーン図** (§0 で 1 PR に集約した場合はチェーン図を省略し、`起点: <起点ブランチ>` の 1 行のみ記す):
```
<起点ブランチ>
  └── feature/xxx (PR1)
        └── feature/xxx-frontend (PR2)
```

**ガイドライン準拠チェック**:
- [x] 各PR 2コミット以内
- [x] 各PR ファイル数 5以下
```

## 判断例

| タスク構成 | PR分割 |
|-----------|--------|
| 全体で5ファイル以下 | **原則 1PR で完結**（分割は依存・レイヤー分離に明確な理由がある時のみ） |
| API 3ファイル + UI 3ファイル | PR1: API, PR2: UI（計6ファイルで上限超のため分割） |
| API 5ファイル + テスト 3ファイル | PR1: API, PR2: テスト |
| 全体で12ファイル（複雑） | 3PRに分割 |
