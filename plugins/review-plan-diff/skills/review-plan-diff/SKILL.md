# review-plan-diff

確定プラン (`/finalize-plan` 出力) と実装後の diff を突き合わせ、実装漏れ・仕様逸脱・計画外差異・スコープ越境を Critical/Major/Minor で判定する検査。

判定を行う `agents/plan-diff-reviewer.md` は、廃止された仕様→PR チェーン自動遷移オーケストレータの一部だった検査を移設したものである。同オーケストレータは実案件 dogfood (2026-07-06) で実装リードタイムが遅すぎる (実装フェーズだけで 5.2 時間) ことを理由に利用者が廃止を決定したが、この突き合わせ検査自体は遅さの原因ではなく、同 dogfood で最も欠陥を検出した部品だった (実装漏れの Critical 2 件 — 計画済みだったテストの未実装 — を検出)。廃止の巻き添えにせず単独起動できる skill として救出した。

## Arguments

`$ARGUMENTS`: 確定プランファイルパス (省略可)

## ワークフロー

### Step 1: 入力の特定

1. **確定プランファイルパス**:
   - `$ARGUMENTS` が指定されていればそれを採用する
   - 省略されている場合、現在のセッションコンテキストに既出のプランファイルパス (典型的には `~/.claude/plans/*.md`、`/finalize-plan` の出力先) があればそれを採用する
   - どちらも無い場合は以下を表示してユーザーに尋ね、返答を待って停止する:
     「突き合わせ対象の確定プランファイルパスを教えてください。」
2. **分析ファイルパス**: プランファイルの拡張子前に `.analysis` を挿入して導出する (例 `feature-xxx.md` → `feature-xxx.analysis.md`)。存在しない場合は Step 3 の起動プロンプトに「無し」と渡し、AC/MECE分析結果との突き合わせは対象外にする。

### Step 2: 比較対象 diff の決定

1. **起点ブランチ**: 以下の優先順で決定する:
   - プランファイルまたは同ディレクトリの `<プラン名>.preflight.md` に起点ブランチの記載があればそれを採用する
   - 記載が無ければユーザーに尋ね、返答を待って停止する:
     「diff の起点ブランチを教えてください (例: develop、main)。」
2. 比較対象は `<起点ブランチ>..HEAD` とする。

### Step 3: レビューエージェント起動

`Task` ツール (`subagent_type: general-purpose`、本リポ標準の dispatch パターン) で `plan-diff-reviewer` を起動する:

```
Task:
  subagent_type: general-purpose
  prompt: |
    あなたはプラン突き合わせレビューエージェントです。
    以下の指示ファイルを読み、その内容に厳密に従ってレビューを実行してください。

    指示ファイル: ${CLAUDE_PLUGIN_ROOT}/skills/review-plan-diff/agents/plan-diff-reviewer.md

    ## 入力
    - 確定プランファイルパス: {Step 1 で特定したパス}
    - 分析ファイルパス: {Step 1 で導出したパス、無ければ「無し」}
    - 対象 diff の取得コマンド: git diff {Step 2 で決定した起点ブランチ}..HEAD

    指示ファイルのワークフローに従い、プラン記載の各項目 (AC・QA-ID・PR分割で定義された変更範囲) と diff を突き合わせてください。
    結果は指示ファイルの「結果出力」フォーマットに厳密に従ってください。
```

### Step 4: 結果報告

エージェントの出力をそのままユーザーへ提示し、総合判定に応じて以下を付す:

- **Critical あり**: 該当行を「未実装の計画項目」として提示し、修正を促す
- **Major/Minor のみ**: 一覧を提示する (修正するかどうかの判断はユーザーに委ねる)
- **乖離なし**: そのまま完了報告する

## 併用推奨 skill

- `/finalize-plan` — 突き合わせ対象の QA-ID カバレッジマトリクス・PR分割計画を供給する
- `/qa-ui` — 本検査の後段で UI 面の検証を行う
